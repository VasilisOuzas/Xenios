# Progress (Version 0.7)
> This is the last experimental version of the application. It is mostly optimized, polished and I've tested it thoroughly.
---
This is the Seventh and last unofficial version of the application. It is working properly for the most part, but more testing needs to be done. The data input/output stream is hosted via .json files that are saved in the `/data` directory.

# Fixes/Upgrades over prior Version 0.3
- Added the option to add extra beds for adults or children in each room, with an additional visual notification for such scenarios.
- Now on there is the option to edit reservations. By selecting to edit a reservation, you can modify all saved data about each room.
- Improved how names are shown on the general matrix/calendar. If a reseration is ling enough, it will display the customer's full name instead of displaying only his surname/last name which was the case on previous versions.
- You can now find and modify easily the reservations, since they are sorted alphabeticaly on the "New Reservation" tab
- (probably) added XENIOS logo in the GUI

 # (Issues) 
- There are no more issues. Although, there is room for improvement. I would like to improve the app more before making the first official version on stuff like code readability, app customizability and UI visuals. I also need to add a real logo in the app before making this official.
